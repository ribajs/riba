# Changelog

All notable changes to Riba.js. Versions from 2.0.0 onwards follow
[Conventional Commits](https://www.conventionalcommits.org/) and are
generated by `release-it` (see [`.release-it.json`](./.release-it.json)).
Older entries were hand-curated and are preserved here for historical
reference.

For upgrade instructions across major versions, see [MIGRATION.md](./MIGRATION.md).

## [2.0.0](https://github.com/ribajs/riba/compare/v1.9.0...v2.0.0) (2026-04-15)

Riba.js 2.0 is a modernization release: ESM-only, Node.js `>=24`, Yarn Berry
v4 (PnP), Vite instead of webpack, Vitest instead of Jest, and Bootstrap 5
as the primary CSS framework via the new `@ribajs/bs5` module. See
[MIGRATION.md](./MIGRATION.md#1x--200) for the full upgrade guide.

### ⚠ BREAKING CHANGES

* **Runtime & toolchain:** Node.js `>=24.0.0` required, ESM-only packages,
  Yarn Berry v4 in PnP mode. npm is no longer supported.
* **Build & test:** Migrated from webpack 4 to Vite and from Jest to Vitest.
* **Browser target:** Modern evergreen browsers only. `core-js`,
  `@types/core-js`, and legacy runtime branches (`attachEvent`, script
  `readyState`, `createTextRange`, `CustomEvent` polyfill) were removed from
  `@ribajs/core` and `@ribajs/utils`.
* **Bootstrap 4 module dropped** in favour of `@ribajs/bs5`. Aspect-ratio
  classnames now follow the Bootstrap 5 `ratio` convention.
* **`@ribajs/photoswipe` replaced** by `@ribajs/bs5-photoswipe`.
* **Packages removed:** `@ribajs/bs4`, `@ribajs/photoswipe`, `@ribajs/vue`,
  `@ribajs/pdf`, `@ribajs/typedoc`, `@ribajs/alosaur` (+ Deno stack),
  `@ribajs/webpack-serve`, `@ribajs/shopify-gulp`, and all backend packages
  (`nest-theme`, `nest-lunr`, `deno-ssr`, `node-graphql-client`) moved to
  a separate backend repository.
* **Packages renamed:** `@ribajs/nest-ssr` → `@ribajs/nest-theme`,
  `@ribajs/deno-ssr` → `@ribajs/deno-node-ssr`.
* **Template engine:** Pug templates replaced by JSX via the new
  `@ribajs/jsx` package.
* **Function-binder signature:** template callbacks no longer receive the
  binder as the first argument; the DOM event is now the first parameter.
* **`this.el` removed from components** — use `this` directly.
* **Formatter rename:** `stripHtml` → `strip-html`.
* **SCSS helper rename in bs5:** `theme-light` / `theme-dark` → `tl` / `td`.
* All deprecated legacy binders were removed from `@ribajs/core`.
* Podlove legacy polyfill loader removed from the podcast web-player
  integration.

  binding declarations

* The `Observer` class was extracted into a new dependency,
  [Sightglass](https://github.com/mikeric/sightglass)

## [0.6.0](https://github.com/ribajs/riba/compare/v0.5.13...v0.6.0) (2013-10-16)

### ⚠ BREAKING CHANGES

* All dependencies now stem from the target object, not the view's scope
  object. Update dependency keypaths accordingly.
* The `prefix` configuration is now an absolute prefix — include `"data"` if
  you want data attributes. Default is `rv`. See
  [MIGRATION.md](./MIGRATION.md#05x--060).

### Features

* Support for multiple adapters through interfaces
* Ships with a built-in `.` adapter using ES5 natives (getters and setters)
* Support for nested keypaths (`user.address:zip`)

### Known caveats of the built-in adapter

* Observes array mutations (`push`, `pop`, `unshift`, etc.) but not direct
  index assignments (`array[3] = 'world'`)
* Cannot subscribe to an array's `length` property; use a formatter
  (`list.items | size`) to access length reactively
